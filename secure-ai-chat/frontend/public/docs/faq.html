はじめに

Secure AI Chat とは何ですか？

Secure AI Chat は、企業向けに設計されたセキュアなAIチャットサービスです。OpenAI GPTやAzure OpenAIを活用し、企業のデータセキュリティ要件を満たしながら、AI技術を安全に利用できます。

主な特徴：

  • 企業グレードのセキュリティ
  • マルチテナント対応
  • カスタムAPI設定
  • テンプレート機能
  • 使用量監視とコスト管理

無料トライアルはありますか？

はい、すべてのプランで14-30日間の無料トライアルをご利用いただけます：

  • Starter/Standard: 14日間
  • Professional/Enterprise: 30日間

トライアル期間中は制限なしでサービスを評価できます。クレジットカード登録も不要です。

どのプランを選べば良いですか？

組織規模と使用量に応じてプランをお選びください：

  • Starter（¥29,800/月）: 10名以下の小規模チーム
  • Standard（¥49,800/月）: 50名以下の中規模企業（推奨）
  • Professional（¥98,000/月）: 200名以下の大企業
  • Enterprise（¥198,000/月～）: 200名以上の大規模組織

迷った場合は、Standardプランから始めることをお勧めします。

アカウント

アカウント作成後のメール確認が必要ですか？

はい、セキュリティのためメールアドレスの確認が必要です。

  1. アカウント作成後、確認メールが送信されます
  2. メール内の確認リンクをクリック
  3. アカウントが有効化されます
注意: 確認リンクは24時間で無効になります。期限が過ぎた場合は、ログイン画面から再送信してください。

パスワードを忘れた場合はどうすれば良いですか？

ログイン画面の「パスワードを忘れた方」から簡単にリセットできます：

  1. ログイン画面の「パスワードを忘れた方」をクリック
  2. メールアドレスを入力
  3. リセット用メールが送信されます
  4. メール内のリンクから新しいパスワードを設定

チームメンバーを招待するにはどうすれば良いですか？

管理者アカウントから簡単にメンバーを招待できます：

  1. ダッシュボードの「ユーザー管理」をクリック
  2. 「新しいユーザーを招待」をクリック
  3. 招待したいメンバーの情報を入力
  4. 招待メールが自動送信されます

招待されたメンバーは24時間以内にアカウントを有効化する必要があります。

使用方法

AIチャットの基本的な使い方を教えてください

シンプルで直感的な操作でAIチャットをご利用いただけます：

  1. ダッシュボードの「新しいチャット」をクリック
  2. メッセージ入力欄に質問や依頼を入力
  3. 送信ボタンをクリックまたはEnterキーを押す
  4. AIからの回答を確認
ヒント: より具体的で明確な指示を与えることで、より良い回答が得られます。

テンプレート機能はどのように使いますか？

テンプレート機能を活用することで、一貫性のある高品質な出力を効率的に生成できます：

  1. 「テンプレート」メニューから適切なテンプレートを選択
  2. 必要な変数（{変数名}）に値を入力
  3. 「生成開始」をクリック
  4. 生成された結果を確認・編集

カスタムテンプレートを作成することで、組織独自のワークフローに対応できます。

チャット履歴は保存されますか？

はい、チャット履歴は安全に保存され、いつでも確認できます：

  • すべてのチャット履歴は暗号化されて保存
  • ダッシュボードの「チャット履歴」から過去の会話を検索・確認
  • データ保持期間は設定で変更可能（30日〜無制限）
  • 必要に応じて個別のチャットを削除可能
セキュリティ: 機密性の高い情報を扱う場合は、データ保持期間を短く設定することをお勧めします。

API設定

システムデフォルトと独自APIの違いは何ですか？

それぞれに異なるメリットがあります：

システムデフォルト:

  • 設定不要ですぐに利用開始
  • 月額固定料金で予算管理が簡単
  • 月間トークン制限あり

独自API（OpenAI/Azure OpenAI）:

  • 使用量に応じた従量課金
  • トークン制限なし
  • コスト最適化が可能
  • 企業のガバナンス要件に対応（Azure OpenAI）

中規模以上の組織では独自APIの使用を推奨します。

OpenAI APIキーはどこで取得できますか？

OpenAI Platform から簡単に取得できます：

  1. https://platform.openai.com にアクセス
  2. OpenAIアカウントでログイン（未登録の場合は新規作成）
  3. 「API Keys」メニューから「Create new secret key」をクリック
  4. 生成されたAPIキーをコピー
  5. Secure AI Chatの設定画面で入力・保存
重要: APIキーは機密情報です。第三者に漏洩しないよう厳重に管理してください。

Azure OpenAI を使用するメリットは何ですか？

Azure OpenAI は企業のガバナンス要件により適した選択肢です：

  • データ主権: データの処理場所を制御可能
  • 企業契約: Microsoft の企業向けSLA
  • コンプライアンス: SOC 2、GDPR等への準拠
  • プライベートネットワーク: VNet統合による高いセキュリティ
  • 統合管理: Azure環境との一元管理

大企業や規制の厳しい業界では Azure OpenAI の使用を推奨します。

請求・支払い

料金の請求はいつ発生しますか？

請求タイミングはプランと支払い方法によって異なります：

無料トライアル期間中:

  • 請求は発生しません
  • クレジットカード登録不要

有料プラン:

  • 月次請求: 毎月同じ日付
  • 年次請求: 契約開始月の同じ日付
  • 使用量ベース（独自API）: 翌月5日

請求書は支払期限の10日前にメールで送信されます。

どのような支払い方法が利用できますか？

企業のニーズに応じて複数の支払い方法をご用意しています：

  • クレジットカード: Visa、Mastercard、JCB、American Express
  • 銀行振込: 月次・年次契約（Standard以上）
  • 請求書払い: 企業向け後払い（Professional以上）
  • 口座振替: 年次契約のみ（Enterprise）

支払い方法の変更は、ダッシュボードの「請求設定」からいつでも可能です。

プランの変更やキャンセルはできますか？

はい、いつでもプランの変更・キャンセルが可能です：

プランアップグレード:

  • 即座に反映
  • 差額は日割り計算で次回請求に加算

プランダウングレード:

  • 次回更新時に反映
  • 現在の請求期間は継続

サービスキャンセル:

  • いつでも可能（違約金なし）
  • データは90日間保持後削除

セキュリティ

データはどのように保護されていますか？

企業グレードのセキュリティでお客様のデータを保護しています：

暗号化:

  • 転送時暗号化: TLS 1.3
  • 保存時暗号化: AES-256
  • データベース暗号化: 透過的データ暗号化

アクセス制御:

  • マルチファクタ認証（MFA）
  • IPアドレス制限
  • 役割ベースアクセス制御（RBAC）

監査:

  • 全操作ログの記録
  • 不正アクセス検知
  • 定期的なセキュリティ監査

チャットデータはAIの学習に使用されますか？

いいえ、お客様のデータは一切学習に使用されません：

  • OpenAI API: エンタープライズ契約により学習除外
  • Azure OpenAI: 設計上、顧客データは学習に使用されない
  • 当社システム: データは処理のみに使用、学習は行わない
透明性: データの使用方法について詳細はプライバシーポリシーをご確認ください。

どのコンプライアンス規格に準拠していますか？

主要なコンプライアンス規格に準拠しています：

  • SOC 2 Type II: セキュリティ・可用性・機密性
  • ISO 27001: 情報セキュリティマネジメント
  • GDPR: EU一般データ保護規則
  • 個人情報保護法: 日本の個人情報保護法
  • PCI DSS: 決済カード業界データセキュリティ基準

コンプライアンス証明書は管理画面からダウンロードできます。

トラブルシューティング

「API接続エラー」が表示される場合はどうすれば良いですか？

API接続エラーの一般的な原因と対処法：

  1. APIキー確認: 設定画面で正しいAPIキーが入力されているか確認
  2. 使用量制限: OpenAI/Azure側の使用量制限に達していないか確認
  3. 接続テスト: 設定画面の「接続テスト」ボタンで動作確認
  4. ネットワーク: 企業ファイアウォールがAPIアクセスをブロックしていないか確認

それでも解決しない場合は、サポートまでお問い合わせください。

AIの応答が遅い・応答がない場合はどうすれば良いですか？

応答が遅い場合の確認事項と対処法：

  1. システム状況: ステータスページで障害情報を確認
  2. プロンプトサイズ: 長すぎる入力は短く分割して送信
  3. サーバー負荷: 時間を空けて再度試行
  4. ブラウザ: ページを再読み込み、別のブラウザで試行

継続的に問題が発生する場合は、システム設定やネットワーク環境の見直しが必要な場合があります。

ユーザーがログインできない場合はどうすれば良いですか？

管理者として確認すべき事項：

  1. アカウント状態: ユーザー管理画面でアカウントが有効か確認
  2. メール確認: メールアドレスが確認済みか確認
  3. 権限設定: 適切な権限が付与されているか確認
  4. パスワード: パスワードリセットを実行

ユーザー側の対処法：

  • ブラウザのキャッシュ・Cookieをクリア
  • プライベートブラウジングモードで試行
  • 別のブラウザまたはデバイスで試行

検索結果が見つかりません

検索キーワードを変更するか、サポートにお問い合わせください

さらにサポートが必要ですか？

解決できない問題がある場合は、お気軽にサポートチームにお問い合わせください
