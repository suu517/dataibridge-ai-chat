はじめに

Secure AI Chat へようこそ！このガイドでは、管理者として効果的にシステムを運用するための手順を詳しく説明します。

このガイドで学べること

  • 初期設定の完了方法
  • ユーザーの招待と管理
  • API設定の最適化
  • セキュリティベストプラクティス
  • 使用量の監視と最適化

管理者の責任範囲

ユーザー管理

チームメンバーの招待、権限設定、アクセス管理を行います。

システム設定

API設定、セキュリティ設定、使用制限の管理を行います。

使用量監視

トークン使用量、コスト、パフォーマンスの監視を行います。

セキュリティ

データ保護、アクセス制御、コンプライアンス遵守を確保します。

初期設定

設定完了の目安

初期設定は通常15-30分程度で完了します。このセクションの手順に従って進めてください。

必須設定項目

  1. メールアドレスの確認

    アカウント作成時に送信された確認メール内のリンクをクリックして、メールアドレスを確認してください。

  2. プロフィール情報の更新

    ダッシュボード右上のプロフィールアイコンから、基本情報を更新してください。

    ダッシュボード → プロフィール → 基本情報を編集

  3. 会社情報の設定

    テナント設定から会社情報を詳細に設定してください。

    • 会社名
    • 業界
    • 従業員数
    • タイムゾーン

  4. 初期ユーザーの招待

    チームメンバーを招待して、協力してシステムを評価できるようにします。

推奨設定項目

推奨設定

  • API設定の選択: 自社のOpenAI/Azure OpenAI APIを設定
  • セキュリティポリシー: パスワードポリシーとセッション設定
  • 通知設定: 使用量アラートと管理者通知
  • 使用制限: ユーザー別・プロジェクト別の制限設定

ユーザー管理

ユーザー招待の手順

  1. 招待リンクの生成

    ダッシュボードの「ユーザー管理」から「新しいユーザーを招待」をクリックします。

  2. 招待情報の入力

    • メールアドレス
    • 氏名
    • 役割（一般ユーザー/管理者）
    • 部署・チーム

  3. 招待メール送信

    システムが自動で招待メールを送信します。招待されたユーザーは24時間以内にアカウントを有効化する必要があります。

一般ユーザー

  • テンプレートの使用
  • 個人設定の変更

管理者

  • ユーザー管理
  • システム設定
  • 使用量監視

注意事項

  • 管理者権限は慎重に付与してください
  • 退職者のアカウントは即座に無効化してください
  • 定期的にユーザー一覧を確認し、不要なアカウントを削除してください

API設定

API設定の選択肢

推奨設定

中規模以上の組織では、コスト最適化のため独自のAPIキーを使用することを強く推奨します。

1. システムデフォルト（簡単開始）

メリット: 即座に利用開始、設定不要

デメリット: 月間トークン制限、コスト予測困難

適用対象: 小規模チーム、評価目的

2. OpenAI API（推奨）

  1. OpenAI APIキーの取得

    https://platform.openai.com でアカウントを作成し、APIキーを生成します。

  2. API設定への入力

    ダッシュボード → API設定 → OpenAI → APIキーを入力 → テスト接続

  3. 使用量制限の設定

    OpenAI側で月間使用量制限を設定することを推奨します。

3. Azure OpenAI（エンタープライズ）

  1. Azureリソースの作成

    Azure PortalでOpenAI Serviceリソースを作成します。

  2. デプロイメントの設定

    GPT-4またはGPT-3.5-Turboモデルをデプロイします。

  3. 接続情報の設定

    • エンドポイントURL
    • APIキー
    • APIバージョン
    • デプロイメント名

API設定テスト

設定完了後は必ず「接続テスト」を実行して、正常に動作することを確認してください。

セキュリティ設定

セキュリティベストプラクティス

重要

企業データを扱う場合、セキュリティ設定は必須です。以下の設定を必ず実施してください。

パスワードポリシー

推奨設定

  • 最小8文字以上
  • 大文字・小文字・数字・記号を含む
  • 辞書攻撃対策
  • 定期的なパスワード変更

セッション管理

  • 自動ログアウト: 30分
  • 同時ログイン制限
  • IPアドレス制限（オプション）
  • デバイス登録制限

データ保護設定

  1. データ暗号化

    保存時暗号化と転送時暗号化が自動で有効になっています。

  2. ログ監査

    全てのユーザー操作とAPIアクセスがログに記録されます。

  3. データ保持ポリシー

    チャット履歴の自動削除期間を設定できます（30日〜無制限）。

コンプライアンス

準拠規格

  • GDPR: EU一般データ保護規則
  • SOC 2: セキュリティ・可用性・機密性
  • ISO 27001: 情報セキュリティマネジメント

監視ダッシュボード

リアルタイムで以下の情報を監視できます：

トークン使用量

  • 月間使用量と制限

  • 時間別トレンド

  • ログイン頻度

アラート設定

  1. 使用量アラート

    月間制限の80%、90%、95%で自動アラートを設定できます。

  2. コストアラート

    予算超過の危険がある場合に通知を受け取れます。

  3. エラーアラート

    API接続エラーや認証エラーを即座に通知します。

レポート機能

定期レポート

  • 週次レポート: 使用量サマリー
  • 月次レポート: コスト分析と最適化提案
  • 四半期レポート: ROI分析とトレンド

テンプレート管理

テンプレートの活用メリット

  • 品質向上: 一貫性のある高品質な出力
  • 効率化: 繰り返し作業の自動化
  • 標準化: 組織全体での統一されたアプローach
  • コスト削減: 最適化されたプロンプトでトークン使用量を削減

テンプレート作成のベストプラクティス

  1. 明確な目的設定

    テンプレートの使用場面と期待する出力を明確に定義します。

  2. 変数の適切な設計

    ユーザーが入力すべき変数を{変数名}形式で定義します。

    例: 「{商品名}の特徴を{ターゲット顧客}向けに説明してください」

  3. プロンプト最適化

    明確で具体的な指示を含め、望ましい出力形式を指定します。

  4. テストと改善

    実際に使用してフィードバックを収集し、継続的に改善します。

トラブルシューティング

よくある問題と解決方法

API接続エラー

症状: "API接続に失敗しました" エラーが表示される

対処法:

  1. API設定画面で接続テストを実行
  2. APIキーが正しいことを確認
  3. 使用量制限に達していないか確認
  4. ネットワーク接続を確認

ユーザーログインできない

症状: 正しい認証情報でもログインに失敗する

対処法:

  1. メールアドレス認証が完了しているか確認
  2. アカウントが無効化されていないか確認
  3. ブラウザのキャッシュとCookieをクリア
  4. パスワードリセットを試行

応答が遅い

症状: AIの応答に時間がかかる

対処法:

  1. システムステータスページを確認
  2. プロンプトの長さを確認（短縮を検討）
  3. API制限に近づいていないか確認
  4. 他のユーザーも同様の問題があるか確認
